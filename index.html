Contrôler un afficheur LCD avec une liaison série

Programme de test en C# et description de la classe SerialELCD162

Afficheur LCD 2 x 16 caractères à commandes séries.

Cet afficheur LCD "OEM" à commandes séries est composé d'un afficheur LCD standard associé à une platine de commande pouvant être commandée par un microcontrôleur au moyen d'ordres séries (8 bits - sans parité - 1 bit de stop - débit binaire 19200b/s ou 4800 b/s). L'afficheur (à processeur Hitachi) dispose d'un réglage de contraste au moyen d'un potentiomètre présent sur la platine de commande.

  • Alimentation: 5V
  • Commande : liaison série asynchrone 8 bits - sans parité - 1 bit de stop - débit binaire 19200b/s ou 4800 b/s
  • Dimensions: 80mm x 36mm
  • Documentation : Comfile ELCD162
  • Distributeur: Lextronic

Carte Netduino plus 2

  • Microcontrôleur : STM32F405RG 32 bits à architecture Cortex-M4 cadencé à 168 MHz.
  • RAM : 100 KB.
  • Flash : 384 KB.
  • Port Ethernet 10Mbps.
  • E/S numériques : 14
  • Entrées analogiques : 6
  • IDE : Microsoft Visual Studio (professionnel ou express)
  • Framework : .NETMF
  • Langage de programmation : C#

  • Site: Netduino
  • Distributeur : Lextronic

Classe SerialELCD162

Utilisation

  1. Créer un objet en utilisant le constructeur SerialELCD162 approprié.
  2. Initialiser la carte avec la méthode Init.
  3. Transmettre des données avec la méthode PutString

Constructeur

Syntaxe Description
SerialELCD162() Initialise un objet "Afficheur LCD série" avec les paramètres: COM2, 19200b/s, pas de parité, 8bits, 1 bit de stop).
SerialELCD162(string com, int baudrate, Parity parite, int bits, StopBits stopb) Initialise un objet "Afficheur LCD série" avec des paramètres de communication personalisés .
  • com: port série sélectionné sur la carte Netduino (SerialPorts.COMx [x=1, 2, 3 ou 4]) Privilégier les ports COM2 et COM3 avec la carte TinkerKit.
  • baudrate: Débit binaire (19200b/s ou 4800b/s)
  • parite: type de parite (System.IO.Ports.Parity.x [x=Even, Mark, None, Odd ou Space])
  • bits: nombre de bits des données (7 ou 8)
  • stopb: nombre de bits de stop (System.IO.Ports.StopBits.x [x=None, One, OnePointFive ou Two])

Méthodes publiques

Syntaxe Description
void ClearScreen() Efface le LCD et positionne le curseur en (0,0).
void CursorOff() Désactivation du curseur.
void CursorOn() Activation du curseur.
void Init() Initialisation de l'afficheur.
A appliquer avant l'envoi des caractères
void PutString(string text) Transmet une chaîne de caractère au LCD.
text: chaîne de caractères codés en ASCII
void SetCursor(byte x, byte y) Place le curseurà la position définie par x (0 à 19 suivant le nombre de caractères de l'écran)
et y (0 à 3 suivant le nombre de lignes).
Si x=0 et y=0 le curseur sera situé en haut à gauche de l'écran.
